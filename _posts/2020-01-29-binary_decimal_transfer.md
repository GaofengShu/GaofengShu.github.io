# 数值二进制和十进制之间的转换

{:toc}

## 起因

近来混迹于[MATLAB 中文论坛](https://www.ilovematlab.cn/forum.php)，帮助解决 matlab 新手各种各样的问题。常见的问题是，他们对于 matlab 计算的不精确结果表示疑惑。比如[这个问题](https://www.ilovematlab.cn/thread-589855-1-1.html)：
> 解线性方程组，解为什么不精确

    clear
    A=[2 3 4;3 5 2;4 3 30];b=[6;5;32];
    x=A\b
    x =
    -13.000000000000064
       8.000000000000036
       2.000000000000005
    x=inv(A)*b
    x =
    -13.000000000000142
       8.000000000000057
       1.999999999999996


准确的解为 `-13,8,2`。很明显，这是由于数值计算的截断误差引起的。数值分析课程介绍过，二进制无法精确表示十进制的 0.1。然而，matlab 的函数 `dec2bin` 并不支持十进制小数，于是我萌生了自己写一个能够支持小数的十进制与二进制的转换函数。

## matlab 内嵌函数 `dec2bin` 和 `bin2dec`

### `dec2bin`

`b = dec2bin(d, n)` 这个函数支持两个输入参数，第一个输入参数 d 为需要转换的十进制数，第二个参数 n 为可选参数，输出二进制结果最短长度。输出参数 b 为二进制字符向量或矩阵。

- d 可以是标量、向量、或矩阵，也可以是字符、字符数组。若 d 为数值类型，将只求 d 的绝对值；若 d 为字符或字符数组，函数内部会将其转换成对应的 ASCII 码值，然后转换成二进制。
  1. **无第二个可选参数**。假设 d 为标量，输出对应的二进制字符串首位为 `1`；若 d 为向量或矩阵，先将输入 d 拉直为列向量，然后依次转换为对应的二进制字符矩阵，输出字符矩阵 b 的列宽由 d 中最大的数决定，假设为 $N$。其余短于 $N$ 的二进制结果，在其前面补足 `0`；
  2. **给定第二个参数 n**。若 $n\le N$，则输出结果列宽为 $N$；若 $n>N$，则所有结果在前面补足足够的 `0`，使得列宽为 n。
- n 只允许标量。

### `bin2dec`

`d = bin2dec(b)` 输入参数 b 是只包含 `0,1` 的字符、字符数组、包含字符数组的元胞矩阵，不支持其他数据类型。输出参数 d 为十进制数列向量。

该函数处理第一步使用 `char` 函数将输入 b 转换成等宽的字符数组。对于那些较短的字符串，`char` 转换后会出现很多空白字符，即 “` `”。第二步去除 ` ` 字符，然后在左边补足相应数目的 `0`。最后根据进制原理转换为十进制数。

## 自编函数 `decimal2binary` 和 `binary2decimal`

### 算法

{% include info.html text="十进制转二进制（小数）" %}

小数部分乘 2 后取整 $\rightarrow$ 小数部分乘 2 后取整 $\rightarrow \cdots \rightarrow$ 直到小数部分为零，或满足其他终止条件，每一步取出的整数，即为二进制小数的结果。伪代码如下：

```
整数小数 d
二进制小数 b
计数器 c=0
while d 不等于 0 或 其他终止条件
	c = c+1
	b(c) = floor(d*2)
	d = d*2 - b
end
```

{% include info.html text="二进制转十进制（小数）" %}

二进制转十进制（小数）算法与整数算法大致相同，将小数点前的每位数字与 $2^0,2^1,2^2,\cdots$ 相乘累加，将小数点后的每位数字与 $2^{-1},2^{-2},2^{-3},\cdots$ 相乘累加，两者相加即为所得的十进制结果。

### `decimal2binary`

和 `dec2bin` 一样，调用方式为 `b = decimal2binary(d, n)`，其中输入参数 d 和 n 均为数值类型。d 为需要转换的十进制数，n 为输出二进制字符数组 b 的列宽。

- d 只允许数值类型，不允许其他数据类型。
  1. **省略输入参数 n**。输出字符数组 d 整数部分列宽由 <u>d 中最大数整数部分对应的二进制数位</u>决定，小数部分在 <u>d 中最长小数部分对应的二进制数位</u>和 <u>16</u> 之间取最小值；
  2. **n 为标量**。整数数位规则不变，小数数位长度为 n；
  3. **n 为向量**。整数数位在 <u>d 中最大数整数部分对应的二进制数位</u>和 n(1) 之间取最大值，小数数位长度为 n(2)。
- n 只允许数值类型。可以是标量，也可以是向量和矩阵，其中的 `inf` 或 `NaN` 会被默认值取代。

具体的输出的二进制字符数位如下表所示：

| 调用方式                   | 整数部分列宽                                       | 小数部分列宽                                                 |
| -------------------------- | -------------------------------------------------- | ------------------------------------------------------------ |
| 省略 n                     | d 中最大数的整数部分对应的二进制数位，假设为 $N_i$ | d 中最长小数部分对应的数位，假设为 $N_d$，列宽为 $\min(N_d, 16)$ |
| n 为标量                   | $N_i$                                              | n                                                            |
| n 为向量，且长度大于等于 2 | $\max(N_i, n(1))$                                  | n(2)                                                         |

### `binary2decimal`

处理过程与 `bin2dec` 完全相同，允许的字符除了 `0,1` 外，还包括小数点 `.`。
